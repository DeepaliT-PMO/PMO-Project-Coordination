# Agile Project Coordination & PMO Reporting

## Project Overview
This project demonstrates my hands-on experience in project coordination and PMO support within an Agile delivery environment. It focuses on structured execution, clear reporting, and stakeholder communication using standard industry tools and documentation.

## Objective
The objective of this project is to showcase practical PMO and coordination skills such as tracking progress, managing risks, preparing status reports, and supporting delivery teams throughout the project lifecycle.

## My Role
In this project, I worked as a Project Coordinator / PMO Support Analyst responsible for coordinating activities across teams, maintaining project documentation, and supporting delivery visibility.

## Tools Used
- Jira (Sprint tracking, task and defect management)
- Confluence (Project documentation, meeting notes)
- MS Excel (Status reports, RAID logs, tracking)
- MS Word (BRD, Meeting Minutes)
- MS PowerPoint (Executive status updates)

## Key Artifacts
The following artifacts are created as part of this project:
- Weekly Project Status Report
- RAID Log (Risks, Assumptions, Issues, Dependencies)
- Meeting Minutes (MoM)
- Business Requirements Document (BRD)
- UAT Test Case & Defect Tracker

## What This Project Demonstrates
- Project coordination and PMO reporting fundamentals
- Stakeholder communication and documentation discipline
- Agile delivery support and tracking
- Risk and issue management
- Business analysis and UAT coordination exposure
